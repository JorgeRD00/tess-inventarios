Opciones para compartir TESS Inventario entre 2 computadoras

Escenario: las dos computadoras están en lugares diferentes y se conectan por internet. Julio 2026.

Antes de ver precios, esto es lo más importante: la app TESS Inventario no guarda los datos dentro de Tauri. Actualmente usa un servidor Node.js + Prisma en la misma computadora y una base de datos SQLite local. Si cada computadora tiene su propio archivo de base de datos, no se sincronizan. Para compartir información, ambas deben conectarse al mismo servidor y la misma base de datos que estén en internet (o en una computadora expuesta a internet).

Resumen rápido

Para dos equipos en diferentes lugares necesitas contratar un “sitio” en internet que aloje el servidor y la base de datos. Estas son las opciones más económicas:

Opción Precio aprox./mes Dificultad Ideal si…
Hetzner VPS Recomendado ~$5 USD (~MXN 90-110) Media Quieres lo más barato y no te importa instalar/configurar.

Explicación de cada opción

1. Hetzner VPS Más barato todo en uno

Es como rentar una computadora pequeña en internet. Ahí instalas el servidor Node.js y PostgreSQL tú mismo.

Ventajas
  • El precio más bajo por servidor + base de datos juntos.
  • Una sola factura y control total.
  • Muy estable y buen rendimiento.
  • No hay límites raros de “sleep” ni conexiones.
Desventajas
  • Tienes que instalar y mantener Node, Postgres y backups.
  • Si algo falla, necesitas a alguien que lo revise.
  • Datacenter en Europa/EE.UU.; algo de latencia a México.

Precio: CPX11 desde ~€4.51/mes (~$5 USD, ~MXN 90-110).

2. Render

Te da la base de datos administrada y puedes subir el backend. Tiene una versión gratuita para el servidor web, pero la base de datos es de pago.

Ventajas
  • Muy fácil de usar, casi todo automático.
  • No tienes que mantener el servidor manualmente.
  • Despliegue desde GitHub.
Desventajas
  • El servidor web gratuito se “duerme” tras 15 min sin uso y tarda en despertar.
  • La base de datos PostgreSQL cuesta desde $7/mes.
  • Para un uso fluido necesitas el servidor de pago.

Precio: Postgres desde ~$7 USD/mes. Servidor web gratis (con inactividad) o ~$7/mes para que no se duerma.

3. Supabase (base de datos) + Render/Fly.io (servidor)

Supabase te da una base de datos PostgreSQL gratuita, pero el backend de TESS debe correr en otro servicio (Render o Fly.io).

Ventajas
  • La base de datos es gratis hasta 500 MB.
  • Supabase administra backups y seguridad.
  • Fly.io o Render pueden ser gratis para el backend.
Desventajas
  • Son dos servicios distintos que administrar.
  • Límite de conexiones (~60 en el plan gratis).
  • El backend gratis se puede dormir (cold start).

Precio: $0 si entras en los límites gratis; si creces, ~$25/mes por Supabase + backend.

4. Neon (base de datos) + Render/Fly.io (servidor)

Similar a Supabase: base de datos Postgres serverless gratuita, backend aparte.

Ventajas
  • 500 MB gratis y pago por uso.
  • Se integra bien con Prisma.
  • No tienes que instalar Postgres.
Desventajas
  • En plan gratuito la base de datos puede pausarse y tardar en arrancar.
  • Necesitas servidor aparte.

Precio: $0 para empezar; servidor aparte desde ~$0 a $7/mes.

5. Railway

Plataforma moderna para desplegar aplicaciones Node + base de datos con pocos clics.

Ventajas
  • Muy fácil de desplegar.
  • Todo en una sola plataforma.
  • Buena experiencia para proyectos pequeños.
Desventajas
  • Ya no tiene plan gratuito; mínimo ~$5/mes y suele subir a ~$10-15/mes.
  • El costo puede crecer si no se controla.

Precio: Desde ~$10-15 USD/mes para backend + base de datos pequeños.

6. Aiven

Base de datos PostgreSQL/MySQL en la nube con un plan gratuito generoso.

Ventajas
  • 5 GB gratis, mucho más que Supabase/Neon.
  • Backups incluidos.
  • Buena para almacenar más datos sin pagar.
Desventajas
  • Solo es la base de datos; backend en otro lado.
  • Plan gratuito con algunos límites de CPU/conexiones.

Precio: $0 hasta 5 GB; luego planes de pago.

7. PlanetScale

Base de datos MySQL serverless. Requiere cambiar Prisma de PostgreSQL a MySQL.

Ventajas
  • 5 GB y 1,000 millones de filas gratis.
  • MySQL compatible.
Desventajas
  • Hay que adaptar el esquema de Prisma (relaciones sin foreign keys).
  • Necesitas backend aparte.
  • Más trabajo inicial.

Precio: $0 hasta 5 GB.

¿Cuál elegir?

Situación Mi recomendación
Quiero lo más barato y tengo a alguien que configura Hetzner VPS (~$5/mes)
Quiero lo más fácil sin instalar nada Render o Railway
Quiero empezar gratis y los datos son pocos Supabase + Render/Fly.io
Quiero más espacio gratis Aiven (5 GB) + servidor aparte

¿Qué cambios requiere la app?

  1. Migrar la base de datos: cambiar schema.prisma de sqlite a postgresql (o MySQL).
  2. Migrar datos: pasar lo que ya tienes en inventory.db a la nueva base de datos.
  3. Desplegar el backend Node.js en el servicio elegido con la nueva URL de base de datos.
  4. Cambiar la URL del frontend (src/services/api.service.ts) de 127.0.0.1:3000 a la dirección del backend en internet.
Conclusión: para dos computadoras en diferentes lugares y lo más económico, mi primera opción es Hetzner VPS (~$5/mes). Si no quieres configurar nada, Render o Railway son más sencillos pero cuestan un poco más. Si quieres empezar sin pagar, Supabase o Neon + Render/Fly te permiten arrancar gratis a cambio de algunas limitaciones.
