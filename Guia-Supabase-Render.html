Guía paso a paso: TESS Inventario en Supabase + Render

Para compartir la app entre 2 computadoras en diferentes lugares por internet.

Resumen de lo que vas a hacer: vas a crear una base de datos en Supabase y un servidor en Render. Ambas computadoras se conectarán al mismo servidor. El proyecto ya fue preparado: ahora usa PostgreSQL y la URL del backend es configurable.

Antes de empezar

  • Tener una cuenta de correo para registrarte en los servicios.
  • Tarjeta de crédito/débito (aunque los planes gratis no cobran, algunos piden verificación).
  • El código del proyecto en tu computadora.

Paso a paso

  1. Crea tu base de datos en Supabase

    1. Entra a supabase.com y crea una cuenta.
    2. Clic en New project, elige una organización y dale un nombre, por ejemplo tess-inventario.
    3. Espera unos segundos a que se cree el proyecto.
    4. Ve a Project Settings (icono de engranaje) → Database.
    5. Busca la sección Connection string y selecciona URI.
    6. Copia la cadena de conexión. Se ve así:
      postgresql://postgres.[ref]:[password]@aws-0-[region].pooler.supabase.com:6543/postgres?pgbouncer=true&connection_limit=1
    7. Guarda esa cadena en un archivo de texto seguro. La necesitarás en el paso 4.
    Tip: usa el puerto 6543 (connection pooler) porque Render abre y cierra conexiones constantemente. Así no agotas el límite de Supabase.

  2. Prepara el proyecto en tu computadora

    El código ya fue ajustado para usar PostgreSQL y una URL de backend configurable. Solo tienes que crear los archivos de variables de entorno.

    1. En la carpeta raíz del proyecto (donde está package.json), crea un archivo llamado .env copiando el contenido de .env.example.
    2. Abre .env y deja la línea de VITE_API_BASE_URL para después (cuando Render te dé la URL).
    3. En la carpeta src-tauri, crea un archivo llamado .env copiando el contenido de src-tauri/.env.example.
    4. Pega la URL de Supabase del paso 1 en la línea de DATABASE_URL. 
      DATABASE_URL=postgresql://postgres.xxxxxx:xxxxxxxx@aws-0-xxxx.pooler.supabase.com:6543/postgres?pgbouncer=true&connection_limit=1

  3. Crea las tablas en Supabase

    Primero borra la carpeta con migraciones viejas de SQLite (si existe), porque ahora usaremos PostgreSQL:

    Remove-Item -Recurse -Force -LiteralPath "src-tauri\prisma\migrations"

    Luego, abre una terminal en la carpeta src-tauri y ejecuta estos comandos uno por uno:

    cd src-tauri
npx prisma migrate dev --name init_postgres
npx prisma db seed

    Esto creará todas las tablas en Supabase y llenará los datos iniciales (roles, usuario admin, marcas, categorías, etc.).

    Si ya tienes datos reales: este paso solo crea datos de ejemplo. Para migrar los datos de tu SQLite actual a Supabase, salta a la sección ¿Cómo paso mis datos actuales? al final.

  4. Sube el proyecto a GitHub

    Render necesita leer el código desde internet. La forma más fácil es subirlo a GitHub:

    1. Crea una cuenta en github.com.
    2. Crea un nuevo repositorio privado llamado tess-inventario.
    3. Descarga GitHub Desktop si no sabes usar Git.
    4. Sube la carpeta del proyecto siguiendo las instrucciones de GitHub Desktop.

    O, si prefieres, copia y pega este comando en la terminal dentro de la carpeta del proyecto:

    git init
git add .
git commit -m "Preparar para Supabase y Render"
git branch -M main
git remote add origin https://github.com/TU_USUARIO/tess-inventario.git
git push -u origin main

  5. Crea el servidor en Render

    1. Entra a render.com y crea una cuenta.
    2. Clic en New → Web Service.
    3. Conecta tu cuenta de GitHub y selecciona el repositorio tess-inventario.
    4. Completa los datos:
      CampoValor
      Nametess-backend
      RegionOregon (EE.UU.) o la más cercana
      Root Directorysrc-tauri
      RuntimeNode
      Build Commandnpm install
      Start Commandnpm start
      PlanFree (o Starter si quieres evitar pausas)
    5. En Environment Variables agrega:
      • DATABASE_URL = la URL de Supabase del paso 1
      • NODE_ENV = production
    6. Clic en Create Web Service.

    Render comenzará a instalar y desplegar. En unos minutos te dará una URL como:

    https://tess-backend.onrender.com
    Importante: con el plan gratis, el servidor se duerme después de 15 min sin uso. La primera vez que abras la app puede tardar 30-60 segundos en responder. Si esto te molesta, puedes pagar el plan Starter (~$7/mes) para que esté siempre activo.

  6. Conecta el frontend al servidor de Render

    1. Abre el archivo .env en la raíz del proyecto.
    2. Cambia la URL de Supabase/localhost por la URL que te dio Render: 
      VITE_API_BASE_URL=https://tess-backend.onrender.com
    3. Guarda el archivo.

  7. Compila la app para instalar en ambas computadoras

    Abre una terminal en la carpeta raíz del proyecto y ejecuta:

    npm run tauri:build

    Esto creará los instaladores dentro de src-tauri/target/release/bundle. Busca el archivo .msi (Windows) o .dmg (Mac).

    Copia ese instalador en ambas computadoras e instálalo.

    Cada vez que cambies la URL del backend (por ejemplo, si Render te da otra URL), debes volver a compilar y reinstalar la app en ambas computadoras.

  8. Prueba que todo funciona

    1. Abre la app en una computadora.
    2. Inicia sesión con admin / admin123.
    3. Crea un producto o una venta.
    4. Abre la app en la otra computadora y verifica que el producto/venta aparece.

¿Cómo paso mis datos actuales?

Si ya tienes información real en el archivo database/inventory.db local, tienes dos opciones:

  • Empezar de cero: el paso 3 crea datos de ejemplo. Luego ingresas tus productos, clientes y ventas manualmente.
  • Migrar los datos: esto requiere ayuda de alguien con conocimientos técnicos. Se puede usar pgloader o exportar las tablas de SQLite a CSV e importarlas en Supabase. Si necesitas esto, pídeselo a quien te ayude con computadoras.

Problemas comunes

  • La app no conecta: revisa que VITE_API_BASE_URL en el archivo .env sea exactamente la URL de Render, incluyendo https:// y sin barra al final.
  • “Error de conexión a la base de datos”: verifica que DATABASE_URL esté correcta en Render y que uses el puerto 6543 (connection pooler) de Supabase.
  • La app tarda en cargar: es normal en el plan gratis de Render; el servidor se duerme y tarda en despertar.
  • Tablas vacías o faltan datos iniciales: asegúrate de ejecutar npx prisma db seed en el paso 3.
Resumen rápido: crea Supabase → crea src-tauri/.env con la URL → corre prisma migrate dev y db seed → sube a GitHub → despliega en Render con Root Directory src-tauri → pega la URL de Render en el .env raíz → compila con npm run tauri:build → instala en ambas PCs.
